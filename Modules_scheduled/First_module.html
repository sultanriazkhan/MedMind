1. High-level module overview

Users upload lab reports (PDF/DOCX/TXT) → receive structured JSON with plain‑English explanations, status (normal/low/high/critical), and possible causes. Fully transparent AI pipeline with BioGPT/Mistral.

Goals

  • 99% extraction accuracy for structured lab names
  • < 3 sec latency for explanation generation (CPU)
  • Support 200+ lab tests using LOINC & medical DB
  • Actionable causes via biomedical literature fine-tuning

Key metrics

  • 98% precision on test extraction
  • 95% user satisfaction on explanation clarity
  • < 0.5% critical misclassification

3. AI processing pipeline
Upload Parsing NER + regex Range compare BioGPT explain JSON output

📄 Document parsing

PyMuPDF (images), pdfplumber (tables), python-docx, fallback text extraction.

🧪 Test extraction

scispaCy (en_ner_bc5cdr_md) + LOINC mapped regex patterns.

🧠 Explanation + causes

BioGPT (microsoft/BioGPT) or Mistral-7B via Ollama prompt: "explain {test} = {value} {unit}, status {status}"

4. Document parsing workflow

  • PDF: try PyMuPDF for text; if low density → pdfplumber table extraction + OCR fallback (Tesseract).
  • DOCX: python-docx extract paragraphs & tables.
  • TXT: raw text cleaning (remove extra spaces).

5. Test extraction workflow

  • NER: scispaCy identifies medical entities (test names).
  • Regex: patterns like (Hemoglobin|Hb)\s*:?\s*([\d.]+)
  • LOINC mapping: link synonyms to standard LOINC codes.

6. Normal range comparison

SQLite table lab_ranges (test_name_clean, age_low, age_high, sex, low, high, unit, critical_low, critical_high). Compare value → status: critical/low/normal/high/critical.

7. Explanation generation

Prompt BioGPT: "Hemoglobin 10.2 g/dL is low. Write one sentence explaining what low hemoglobin means." Cached for common tests.

8. Possible cause detection

Two-tier: rule-based (if low hemoglobin → ["iron deficiency","chronic disease","blood loss"]) + BioGPT/Mistral refinement. Causes stored in causes_db keyed by test & direction.

15. Key engineering challenges
  • Messy formats: scanned PDFs, handwritten tables → OCR with Tesseract + layout analysis.
  • Embedded tables: pdfplumber + heuristics to extract test-name/value/unit columns.
  • NER accuracy: fine-tune scispaCy on 500 annotated lab reports.
  • Range ambiguity: different labs use different reference ranges – store multiple ranges per test (age/sex).
16. Testing strategy
  • Unit: parser fixtures (PDF, DOCX, TXT samples).
  • Extraction: compare NER output with annotated ground truth (accuracy > 95%).
  • Classification: 1000+ synthetic test values, validate status correctness.
  • API integration: pytest + upload real files, assert JSON structure.
  • Load: 50 concurrent uploads, measure P95 latency.
