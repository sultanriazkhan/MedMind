01

Module Overview

The Lab Reports Explainer is the flagship feature of MedMind. It transforms unstructured medical documents into structured, human-readable health insights — removing the barrier between raw diagnostic data and patient understanding.

📄

Document Ingestion

Accepts PDF, DOCX, and TXT lab reports via drag-and-drop or paste. Parses and normalizes raw text using PyMuPDF, pdfplumber, and python-docx.

🔬

Test Extraction

Uses scispaCy NER + regex to locate and extract test names, numeric values, and units from heterogeneous lab report layouts.

📊

Range Classification

Compares each test against a LOINC-mapped SQLite reference database. Classifies results as Normal, Low, High, or Critical.

🧠

AI Explanation

BioGPT or Mistral 7B via Ollama generates plain-English explanations and suggests possible causes for every abnormal result.

⚡

Structured Output

Returns a clean JSON payload: test list, statuses, explanations, causes, a summary string, and a total risk count.

🔒

Secure & Private

Reports stored per-user in SQLite with JWT auth. No PHI leaves the server. Files processed in-memory and discarded.

Stage 1–2 · Document Parsing Workflow

PDF

PyMuPDF + pdfplumber

PyMuPDF handles standard PDFs. pdfplumber is used as a fallback for scanned or table-heavy PDFs. Tables are extracted as structured grids.

DOCX

python-docx

Extracts paragraphs, tables, and runs. Handles merged cells and multi-column tables that are common in DOCX lab reports.

TXT / Paste

Plain Text

Raw text is accepted directly. Line-by-line parsing applies regex patterns without any document library overhead.

Stage 3 · Test Extraction Workflow

services/extractor.py
# Dual-strategy extraction: NER first, then regex fallback

import spacy
import re

nlp = spacy.load("en_core_sci_sm")   # scispaCy biomedical NER

# Regex: captures  "Test Name    Value   Unit   Range"
TEST_PATTERN = re.compile(
    r"(?P<name>[A-Za-z][A-Za-z\s\(\)/]+?)\s+"
    r"(?P<value>\d+\.?\d*)\s*"
    r"(?P<unit>[a-zA-Z/%]+(?:/[a-zA-Z]+)?)\s*"
    r"(?P<range>\d+\.?\d*\s*[-–]\s*\d+\.?\d*)?",
    re.MULTILINE
)

def extract_tests(text: str) -> list[dict]:
    results = []
    doc = nlp(text)

    # Pass 1: NER entities labeled as LAB_VALUE
    ner_spans = {ent.text: ent.label_ for ent in doc.ents}

    # Pass 2: Regex for precise value + unit extraction
    for match in TEST_PATTERN.finditer(text):
        results.append({
            "name":  match.group("name").strip(),
            "value": float(match.group("value")),
            "unit":  match.group("unit"),
        })

    return results

Stage 4 · Normal Range Comparison Logic

Classification Rules

  • Query SQLite lab_ranges by test name (fuzzy match with difflib)
  • If value < min_range → status = Low
  • If value > max_range → status = High
  • If value > 2× max or < 0.5× min → status = Critical
  • Otherwise → status = Normal
  • No match found → status = Unknown

Status Badge Reference

Normal Value within reference interval
Low Value below minimum threshold
High Value above maximum threshold
Critical Value dangerously out of range

Stage 5 · Explanation Generation Pipeline

For each extracted test, a structured prompt is built and passed to the AI model. Two model options are supported — BioGPT (CPU-friendly, offline) and Mistral 7B via Ollama (richer output, requires GPU or local server).

services/explainer.py — Prompt Template
PROMPT_TEMPLATE = """
You are a medical assistant AI explaining lab results to a non-medical patient.

Test: {name}
Value: {value} {unit}
Status: {status}
Reference Range: {normal_range}

Explain in 2-3 simple sentences what this test measures and what the result means.
Then list 3 possible medical causes for this result (if abnormal).
Respond as JSON: {{"explanation": "...", "causes": ["...", "...", "..."]}}
"""

def generate_explanation(test: dict) -> dict:
    prompt = PROMPT_TEMPLATE.format(**test)

    # Option A: BioGPT (transformers)
    # from transformers import pipeline
    # pipe = pipeline("text-generation", model="microsoft/BioGPT")

    # Option B: Mistral via Ollama (local API)
    response = requests.post("http://localhost:11434/api/generate", json={
        "model": "mistral",
        "prompt": prompt,
        "stream": False
    })

    return json.loads(response.json()["response"])

api/routes/lab.py — Core Upload Route
from fastapi import APIRouter, UploadFile, File, Depends, HTTPException
from services.parser import parse_document
from services.extractor import extract_tests
from services.comparator import compare_ranges
from services.explainer import generate_explanations
from db.reports import save_report

router = APIRouter(prefix="/api/lab")

ALLOWED_TYPES = {"application/pdf", "application/vnd.openxmlformats...", "text/plain"}

@router.post("/upload")
async def upload_report(file: UploadFile = File(...), user=Depends(get_current_user)):
    if file.content_type not in ALLOWED_TYPES:
        raise HTTPException(400, "Unsupported file type")
    doc_id = await save_document(file, user.id)
    return {"doc_id": doc_id, "status": "uploaded"}

@router.post("/analyze/{doc_id}")
async def analyze_report(doc_id: str, user=Depends(get_current_user)):
    raw_text = await parse_document(doc_id)
    tests    = extract_tests(raw_text)
    compared = await compare_ranges(tests)
    result   = await generate_explanations(compared)
    await    save_report(doc_id, user.id, result)
    return   result

Key Engineering Challenges

🗂️ Messy & Inconsistent Lab Report Formats

Problem: Labs produce PDFs with wildly different layouts — multi-column tables, merged cells, scanned images, or embedded watermarks. A single regex cannot cover all formats.

Solution: Layer two parsers (PyMuPDF for structure, pdfplumber for table coordinates). Detect if a page is scanned (low text density → flag for manual review). Store the raw extracted text for debugging.

📊 Table Extraction Accuracy

Problem: Lab results are almost always in tables. PDF table extraction frequently mis-aligns columns — the test name and value end up in the wrong cells.

Solution: Use pdfplumber's explicit table-detection with horizontal line anchors. Post-process extracted rows to validate that numeric values follow known test-name strings. Fall back to regex on the raw text if table extraction yields less than 3 structured rows.

🔬 Medical NER Accuracy & Coverage

Problem: scispaCy's en_core_sci_sm model may not recognize all test names (regional labs, non-standard abbreviations, vernacular names).

Solution: Maintain a custom entity ruler (spaCy's EntityRuler) seeded with 300+ test name patterns from the LOINC dataset. Combine this with the ML NER for maximum coverage.

🧠 AI Model Latency on CPU

Problem: BioGPT on CPU can take 30–90s per explanation. Generating explanations for 20 tests in series would be unacceptably slow.

Solution: Batch all abnormal tests into a single multi-test prompt. Run explanations for normal tests from the causes_db SQLite cache rather than the AI model. Consider async background processing with WebSocket status push.

🔍 Test Name Matching (Fuzzy Lookup)

Problem: "Hgb", "Hemoglobin", "Hb", and "Haemoglobin" all refer to the same test but won't match a direct string lookup.

Solution: Store an aliases JSON array in lab_ranges. On lookup, try exact match → alias match → difflib.get_close_matches(cutoff=0.8). Log unmatched test names for iterative dataset improvement.

🔐 PHI & Medical Data Privacy

Problem: Lab reports contain Personally Identifiable Information. Storing or transmitting PHI insecurely is a serious liability.

Solution: Process files entirely in-memory. Delete uploaded files immediately after text extraction. Store only the result JSON (no raw file text). Use HTTPS and JWT expiry of 24h. Add a per-report delete API for data rights compliance.

Testing Strategy

A four-layer testing pyramid ensures quality at every stage of the pipeline.

🔧

Unit Tests — Services

Test each service module in isolation. Parser tests use pre-collected PDF fixtures. Extractor tests validate NER and regex on labeled sample text. Comparator tests cover all four status branches and edge cases (exactly at boundary).

pytest pytest-asyncio mock / monkeypatch
🌐

API Integration Tests

Use FastAPI's TestClient (HTTPX) to call every route. Verify status codes, response schemas (Pydantic), JWT rejection, and unsupported file type errors. Test upload + analyze in sequence with a fixture PDF.

pytest httpx TestClient faker
🖥️

End-to-End Tests (E2E)

Playwright scripts automate the full user journey: visit landing → sign up → upload PDF → wait for processing → verify results table contains expected test names and statuses.

Playwright pytest-playwright Headless Chromium
📈

Extraction Accuracy Evaluation

Manually labeled a golden set of 20 real lab reports with ground-truth test names, values, and statuses. Run extractor + comparator and measure Precision, Recall, and F1. Target: F1 ≥ 0.85.

sklearn.metrics Precision / Recall / F1 JSON fixtures
tests/test_extractor.py — Sample Test
import pytest
from services.extractor import extract_tests

SAMPLE_TEXT = """
Test Name           Result    Unit    Reference Range
Hemoglobin          10.2      g/dL    12.0 - 17.5
Glucose             98        mg/dL   70 - 125
WBC Count           11.4      10^3/μL 4.5 - 11.0
"""

def test_extract_hemoglobin():
    tests = extract_tests(SAMPLE_TEXT)
    hgb = next(t for t in tests if "hemoglobin" in t["name"].lower())
    assert hgb["value"] == 10.2
    assert hgb["unit"]  == "g/dL"

def test_extract_all_three():
    tests = extract_tests(SAMPLE_TEXT)
    assert len(tests) == 3

def test_empty_text():
    tests = extract_tests("")
    assert tests == []
